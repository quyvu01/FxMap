# Changelog v2.0.0

All notable changes in this release.

## [2.0.0] - 2026-04-05

> **Breaking release.** Several public APIs have changed or been removed. See the [Migration Guide](#migration-guide) section below before upgrading.

---

## Breaking Changes

### 1. `UseDistributedKey` string overload now requires a namespace

**Before (1.x):**
```csharp
UseDistributedKey<UserDistributedKey>("UserKey");
```

**After (2.0):**
```csharp
UseDistributedKey<UserDistributedKey>("UserKey", "MyService");
```

The second `namespace` parameter is required. The runtime cache key changes from a plain `string` to a `(key, namespace)` tuple, scoping all dynamically-emitted types to `{namespace}.{key}`. This prevents name collisions when multiple services use the same key string.

Validation is enforced at configuration time: both `key` and `namespace` must match `^[A-Za-z][A-Za-z0-9_]*$` or startup will throw.

---

### 2. `AddHandlersFromNamespaceContaining` removed from `MapConfigurator`

Pipeline behaviors are now the only handler registration mechanism. Remove any call to `AddHandlersFromNamespaceContaining` in your `AddFxMap(...)` setup — it no longer exists.

---

### 3. gRPC extension renamed

`MapFxMapGrpcService()` → **`MapFxMapperGrpc()`**

```csharp
// Before
app.MapFxMapGrpcService();

// After
app.MapFxMapperGrpc();
```

---

### 4. gRPC delegate renamed

`GetFxMapResponseFunc` → **`GetMapperResponseFunc`**

---

### 5. `GetAssemblyName()` extension removed

Use the standard `AssemblyQualifiedName` property directly.

---

### 6. Removed members

| Member | Reason |
|--------|--------|
| `MapConfigurator.AddHandlersFromNamespaceContaining` | Pipelines are the only registration path |
| `TypeExtensions.IsInterfaceOrConcreteClass` | Unused internal helper |
| `TypeExtensions.IsFSharpType` | Unused internal helper |
| `MessageRequestWrapped.Subject` | Replaced by string-based routing |
| `GetAssemblyName()` extension | Use `AssemblyQualifiedName` |

---

## New Features

### Namespace-scoped string distributed keys

String-based `UseDistributedKey` calls are now scoped to a `{namespace}.{key}` pair. The dynamic module used to emit these types is renamed to `DistributedMappingModule`. This guarantees uniqueness across services that may happen to share the same key name string.

`IFluentEntityConfig` gains a new `DistributedNamespace` property surfaced through `EntityConfigureOf<T>` and `ProfileOf<T>`.

---

### Centralized type lookup (`GetDistributedTypeData`)

A new `DistributedTypeData` record and `ConfigurationExtensions.GetDistributedTypeData` replace the per-transport `Type.GetType()` + dictionary lookup pattern. Results are cached in a `ConcurrentDictionary`.

Dynamic types emitted at runtime cannot be re-resolved by name via `Type.GetType()`. All transports now go through this single method instead of duplicating the resolution logic.

```csharp
// All transports resolve types through:
var typeData = config.GetDistributedTypeData(key, ns);
```

---

### `DistributedMapException` type

A new typed exception class `DistributedMapException` is available for catching mapping-specific runtime failures.

---

## Improvements

### gRPC transport is now fully string-based

The gRPC discovery and dispatch pipeline operates entirely on `AssemblyQualifiedName` strings instead of `Type[]`. `DistributedKeysProbe`, `GetMapperResponseFunc`, and `GetDistributedKeysByHost` all accept and return `string[]`. This makes the gRPC transport compatible with dynamically-emitted types that have no fixed assembly name.

### Primary constructor migration

`SqsRequestClient`, `NatsRequestClient`, and `RabbitMqRequestClient` have been migrated to C# primary constructors, removing boilerplate field assignment.

### Sample: gRPC-first wiring

Service2 and Service3 in the sample now expose Kestrel HTTP/2 endpoints and call `MapFxMapperGrpc()`. Service1 connects via `AddGrpcClients`. NATS/RabbitMQ/Kafka/SQS registrations are commented out but remain available as reference.

## Migration Guide

### Step 1 — Add namespace to all string `UseDistributedKey` calls

Search for `UseDistributedKey(` and add a second argument for the namespace:

```csharp
// Find:
UseDistributedKey<TKey>("SomeKey");

// Replace with:
UseDistributedKey<TKey>("SomeKey", "YourServiceName");
```

### Step 2 — Remove `AddHandlersFromNamespaceContaining`

```csharp
// Remove this line:
cfg.AddHandlersFromNamespaceContaining<IAssemblyMarker>();
```

### Step 3 — Rename gRPC extension calls

```csharp
// Find:
app.MapFxMapGrpcService();

// Replace with:
app.MapFxMapperGrpc();
```

### Step 4 — Update any direct delegate references

```csharp
// Find: GetFxMapResponseFunc
// Replace with: GetMapperResponseFunc
```
